# Dev1 已打开页面复核优化计划

## 目标

当网页快照抓取队列的当前列表完全来自已打开的浏览器 Tab 时，不再要求用户点击“在新窗口打开”。用户可以在原浏览器窗口里检查页面，然后直接点击“提交复核”，再点击“开始”抓取。混合队列保持原来的复核窗口流程。

## 范围

1. 只优化纯已打开 Tab 列表。
2. 当前列表中任意一项不是 `useExistingTab + existingTabId`，或已经属于复核窗口 Tab，则走原来的三步流程。
3. 选取域名、子域名、书签树后再混入已打开页面时，第一版不做部分复用，仍统一走复核窗口。
4. 已打开页面的抓取继续复用现有 `existingTabId`，不新开窗口。

## 实施步骤

1. 增加前端判断函数：当前列表全部是用户已有 Tab、且没有 `reviewWindowId` 时，进入 `existing-tab-review` 模式。
2. 在复核流程 UI 中识别该模式：隐藏“在新窗口打开”，把“提交复核”改成第 1 步，“开始”改成第 2 步。
3. 在队列表格状态列中给这类条目显示蓝色标记，如“活跃页面”，让用户知道这些 URL 会直接复用已有 Tab。
4. 修改“提交复核”：该模式下不要求 `reviewWindowId`，改为校验所有 `existingTabId` 是否仍然可用，校验通过后把当前列表标记为已复核。
5. 修改“开始”：该模式下不自动创建复核窗口；若未提交复核则提示用户先提交，已提交后直接进入抓取。
6. 队列行点击时跳转到对应窗口和 Tab；若队列项没有绑定 `existingTabId`，则按 URL 在已打开 Tab 中查找并跳转。
7. 保留原复核窗口逻辑：非纯已打开 Tab 列表继续要求“在新窗口打开 -> 提交复核 -> 开始”。
8. 跑 `node --check` 和 `git diff --check`，确认语法和补丁格式正常。

## 风险控制

1. 不改变后台 MHTML 导出格式和文件夹直出逻辑。
2. 不把混合队列拆成部分复用，避免队列状态和复核窗口状态互相污染。
3. 不主动关闭、激活或 reload 用户已有 Tab；只有用户点击队列行时才聚焦对应页面。
4. 提交复核前只做 Tab 可用性校验，避免页面已关闭后仍进入抓取。
