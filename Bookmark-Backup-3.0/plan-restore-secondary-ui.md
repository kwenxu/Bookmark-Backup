# 恢复二级 UI 改造计划书（`快照 / 变化`）

更新时间：2026-03-01

## 1. 背景

当前恢复二级 UI 中，不同类型（多版本、覆盖、手动导出）的入口与动作语义存在混合：

- `快照恢复` 与 `变化导入` 在同一层逻辑中耦合。
- 用户在某些场景下容易误触“恢复”而非“导入合并”。
- 手动导出模式有 HTML 全兼容诉求，不希望被额外分层影响。

本计划目标是：在不破坏现有扫描/索引规则的前提下，完成“动作语义分层”。

---

## 2. 目标

1. 仅在 `多版本` 与 `覆盖` 下增加二级切换：`快照 / 变化`。
2. `变化` 页只保留一个动作：`导入合并`。
3. `手动导出` 保持现有 HTML 全兼容直通逻辑，不新增 `快照 / 变化` 分层。
4. 保留并兼容现有“索引 / 非索引”机制（仅在 `多版本 + 快照` 场景出现）。

---

## 3. 范围与非范围

### 3.1 本次范围

- 恢复二级 UI 的展示层与交互层重构。
- 动作入口重排（`导入合并` 迁移到 `变化` 子页）。
- 与现有恢复数据模型的最小兼容改造。

### 3.2 非范围

- 不改云端/本地扫描协议。
- 不改索引文件格式（`版本化/备份历史log.md`）。
- 不改手动导出文件解析与 HTML 兼容核心逻辑。

---

## 4. 交互规则（最终形态）

| 一级类型 | 二级切换 | 列表内容 | 可执行动作 |
|---|---|---|---|
| 多版本 | 快照 | 版本快照列表 | 预览 / 恢复 |
| 多版本 | 变化 | 变化工件入口 | 仅导入合并 |
| 覆盖 | 快照 | 覆盖快照列表 | 预览 / 恢复 |
| 覆盖 | 变化 | 变化工件入口 | 仅导入合并 |
| 手动导出 | 不显示此切换 | 现有兼容列表 | 保持现有逻辑 |

补充规则：

- `索引 / 非索引` 仅在 `多版本 + 快照` 显示。
- `变化` 子页下禁止普通恢复动作按钮，仅保留 `导入合并`。

---

## 5. 状态模型设计

建议状态字段（前端）：

- `restorePrimaryType`: `versioned | overwrite | manual_export`
- `restoreSubMode`: `snapshot | changes`
- `versionedIndexView`: `indexed | non_indexed`

约束：

- 当 `restorePrimaryType === manual_export`：忽略 `restoreSubMode`（不渲染切换）。
- 当 `restoreSubMode === changes`：隐藏快照列表与快照动作区。

---

## 6. 实施步骤

### Step 1：UI 骨架扩展

- 在恢复二级 UI 顶部（类型切换附近）新增 `快照 / 变化` 分段。
- 显示条件：`restorePrimaryType` 为 `versioned` 或 `overwrite`。

### Step 2：渲染分支拆分

- 将当前二级 UI 渲染逻辑拆为两条主分支：
  - `renderSnapshotMode(...)`
  - `renderChangesMode(...)`
- `manual_export` 继续走既有直通渲染，不进入上述分支。

### Step 3：动作绑定重排

- `changes` 分支仅挂载 `导入合并`。
- 移除/隐藏 `changes` 分支中的“预览/恢复”按钮。

### Step 4：多版本索引分层保持

- 在 `versioned + snapshot` 继续显示 `索引项 / 非索引项`。
- 在 `versioned + changes` 隐藏索引分层开关。

### Step 5：兼容与回归

- 验证本地文件模式、手动导出模式、云端模式入口不回退。
- 验证 `导入合并` 在 `changes` 分支可单独工作。

---

## 7. 验收标准（Checklist）

- [ ] 多版本可切换 `快照 / 变化`。
- [ ] 覆盖可切换 `快照 / 变化`。
- [ ] 手动导出不显示该切换。
- [ ] `变化` 页仅出现 `导入合并`。
- [ ] `多版本 + 快照` 下保留 `索引 / 非索引`。
- [ ] `多版本 + 变化` 不显示 `索引 / 非索引`。
- [ ] 原有快照恢复链路不受影响。

---

## 8. 风险与防护

### 风险

- 切换状态与旧按钮显隐逻辑冲突，导致 UI 残留。
- `导入合并` 的可见性改动误伤手动导出路径。

### 防护

- 使用单一入口 `renderRestoreSecondaryUI()` 做全量重绘（避免局部叠加）。
- 针对三类主类型分别做可见性断言（开发期日志 + 手工回归）。

---

## 9. 执行顺序建议

1. 先改 UI 状态与显示条件。
2. 再拆分动作绑定与按钮显隐。
3. 最后做联调回归（云端1/云端2/本地、目录模式/文件模式）。

- `popup.html`
