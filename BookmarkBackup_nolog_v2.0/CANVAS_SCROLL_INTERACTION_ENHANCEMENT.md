# 书签画布滚动和拖动交互优化

## 更新日期
2025-11-17

## 概述
本次更新对书签画布（Canvas）的滚动和拖动交互进行了三项重要优化，提升了用户体验的流畅度和自然度。

## 新增功能

### 1. 双指滑动时栏目的防抖动屏蔽
**问题描述：**
在使用触控板双指滑动画布时，如果鼠标光标停留在栏目（永久栏目、书签型临时栏目、空白栏目等）内部，会意外触发栏目的纵向滚动，导致画布滚动和栏目滚动同时发生，造成体验混乱。

**解决方案：**
- 新增 `touchpadState` 状态管理，追踪画布级别的双指滑动状态
- 当检测到触控板双指滑动时，自动标记为"画布滚动模式"
- 在此模式下，栏目内部的滚动事件被拦截，统一由画布处理
- 滚动停止300ms后自动恢复栏目内部的滚动功能

**实现细节：**
```javascript
// 状态追踪
touchpadState: {
    isScrolling: false,      // 是否正在画布级别的滚动
    lastScrollTime: 0,       // 上次滚动时间
    scrollTimeout: null      // 超时定时器
}
```

**用户体验：**
- ✅ 双指滑动画布时，不会误触栏目内的滚动
- ✅ 停止双指滑动后，栏目内的滚动功能立即恢复
- ✅ 保留了栏目内原有的双指纵向滚动功能

---

### 2. 滚动结束时的拖尾阻尼延续动作
**问题描述：**
之前的滚动实现在用户停止滑动后立即停止，没有惯性效果，显得生硬不自然。

**解决方案：**
- 新增 `inertiaState` 惯性状态管理
- 记录滚动时的速度（delta值）
- 滚动停止时，基于最后的速度启动惯性滚动动画
- 应用阻尼系数（damping = 0.92）逐帧减速
- 速度低于阈值时平滑停止

**实现细节：**
```javascript
// 惯性状态
inertiaState: {
    velocityX: 0,            // X轴速度
    velocityY: 0,            // Y轴速度
    isActive: false,         // 是否正在惯性滚动
    animationId: null,       // 动画帧ID
    lastDeltaX: 0,          // 最后的X增量
    lastDeltaY: 0,          // 最后的Y增量
    lastTime: 0             // 最后的时间戳
}
```

**参数调优：**
- `inertiaMultiplier: 1.2` - 初始速度放大系数，让惯性更明显
- `damping: 0.92` - 阻尼系数，控制减速快慢
- `stopThreshold: 0.1` - 停止阈值，速度低于此值时停止
- `velocityThreshold: 0.5` - 启动阈值，速度太小不触发惯性

**用户体验：**
- ✅ 滚动停止后有自然的拖尾效果
- ✅ 减速过程平滑，不会突然停止
- ✅ 适用于纵向、横向和四向滚动
- ✅ 仅在触控板滚动时启用（鼠标滚轮不启用）

### 3. 拖动栏目到边缘时自动滚动画布
**问题描述：**
在拖动栏目（永久栏目或临时栏目）时，如果需要将其移动到当前可视区域外，用户必须先停止拖动、滚动画布、再继续拖动，操作繁琐。

**解决方案：**
- 新增 `autoScrollState` 自动滚动状态管理
- 在拖动过程中实时检测鼠标位置
- 当鼠标接近画布边缘（50px范围内）时，自动触发画布滚动
- 滚动速度根据鼠标距离边缘的距离动态调整（越近越快）
- 拖动的栏目位置同步更新，保持跟随鼠标

**实现细节：**
```javascript
// 自动滚动状态
autoScrollState: {
    intervalId: null,        // 动画帧ID
    velocityX: 0,           // X轴滚动速度
    velocityY: 0,           // Y轴滚动速度
    isActive: false         // 是否正在自动滚动
}
```

**参数配置：**
- `edgeThreshold: 100` - 触发边缘（像素）**（已优化：从50增加到100）**
- `maxSpeed: 20` - 最大滚动速度 **（已优化：从15增加到20）**
- `minSpeed: 2` - 最小滚动速度 **（新增）**
- `smoothing: 0.15` - 速度平滑系数 **（新增：使用线性插值避免抖动）**
- 速度计算：使用三次方缓动函数（easeInCubic）实现平滑加速
  - `easedRatio = ratio³`
  - `speed = minSpeed + (maxSpeed - minSpeed) × easedRatio`

**用户体验：**
- ✅ 拖动栏目到边缘时，画布自动滚动
- ✅ 支持横向、纵向和斜向自动滚动
- ✅ 滚动速度随距离动态调整，操作精准
- ✅ 拖动元素位置实时同步，不会产生位移偏差
- ✅ 释放鼠标后自动停止滚动
- ✅ **（优化）识别范围扩大到100px，更容易触发**
- ✅ **（优化）使用速度插值，滚动过程完全平滑无抖动**
- ✅ **（优化）启动时平滑加速，停止时平滑减速**

## 边缘自动滚动优化详解（2025-11-17 更新）

针对第三个功能进行了深度优化，解决了抖动问题并提升了触发灵敏度。

### 优化点1：识别范围扩大
- **原参数：** `edgeThreshold: 50px`
- **新参数：** `edgeThreshold: 100px`
- **效果：** 从边缘100px范围内就开始触发，更容易激活自动滚动

### 优化点2：速度平滑过渡（核心反抖动优化）
**问题：** 原实现直接使用计算的速度，导致速度突变产生抖动

**解决方案：** 引入速度插值（Linear Interpolation）
```javascript
// 状态分离：当前速度 vs 目标速度
state.velocityX (当前实际)  →  state.targetVelocityX (目标期望)

// 每帧平滑过渡（线性插值）
state.velocityX += (state.targetVelocityX - state.velocityX) × smoothing
```

**参数调优：**
- `smoothing: 0.15` - 平滑系数
  - 值越小越平滑，但响应越慢
  - 值越大响应越快，但可能抖动
  - **0.15 是经过测试的最佳平衡点**

**效果：** 即使鼠标位置剧烈变化，实际滚动速度也会平滑过渡，完全消除抖动

### 优化点3：缓动函数优化速度曲线
**原实现：** 线性速度变化
```javascript
speed = maxSpeed × ratio  // 线性，变化不够自然
```

**新实现：** 三次方缓动（easeInCubic）
```javascript
const easeInCubic = (t) => t × t × t;
easedRatio = easeInCubic(ratio);
speed = minSpeed + (maxSpeed - minSpeed) × easedRatio;
```

**曲线特性：**
- 进入边缘区域时：慢速启动（ratio=0.8 → easedRatio=0.512）
- 越靠近边缘：加速越快（ratio=0.9 → easedRatio=0.729）
- 最接近边缘：接近最大速度（ratio=1.0 → easedRatio=1.0）
- 始终保持 `minSpeed=2` 的基础速度，避免边缘区外侧太慢

### 优化点4：启动平滑化
**首次触发时：**
```javascript
// 不是从0开始，而是从目标速度的50%开始
velocityX = targetVelocityX × 0.5
velocityY = targetVelocityY × 0.5
```
避免了从静止到高速的突变，提供更自然的启动体验。

### 综合效果
- 🚀 **识别范围翻倍：** 从50px增加到100px
- 🎯 **完全消除抖动：** 速度插值 + 缓动函数双重保障
- 🌊 **丝滑体验：** 启动、加速、减速全程平滑
- ⚡ **响应迅速：** 0.15的平滑系数确保快速响应

## 技术实现要点

### 性能优化
1. **RAF（requestAnimationFrame）驱动**
   - 所有动画使用RAF，确保60fps流畅度
   - 自动与浏览器渲染周期同步

2. **快速平移模式**
   - 滚动/拖动时使用 `transform` 直接操作
   - 停止时才切换到CSS变量模式
   - 避免频繁的样式重计算

3. **轻量级滚动条更新**
   - 滚动时只更新 `transform`，不重新计算边界
   - 停止后才进行完整更新

### 状态管理
所有新增状态都集中在 `CanvasState` 对象中：
- `touchpadState` - 双指滑动状态
- `inertiaState` - 惯性滚动状态
- `autoScrollState` - 自动滚动状态

### 事件协调
1. **滚动事件链**
   ```
   wheel事件 → 记录速度 → 取消惯性 → 更新位置
   → 滚动停止 → 启动惯性 → RAF动画 → 平滑停止
   ```

2. **拖动事件链**
   ```
   mousemove → 更新位置 → 检测边缘 → 启动自动滚动
   → RAF动画 → 同步拖动元素 → mouseup → 停止自动滚动
   ```

## 已知限制

1. **触控板检测**
   - 基于 `deltaMode` 和 `delta` 值大小判断
   - 某些设备可能误判，但影响不大

2. **惯性滚动仅触控板**
   - 鼠标滚轮不启用惯性（因为滚轮本身是离散的）
   - 触摸屏可能效果不一致

3. **边缘自动滚动阈值固定**
   - 当前固定50px，未来可考虑配置化
